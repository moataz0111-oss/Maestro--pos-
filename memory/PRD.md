# PRD - نظام إدارة المطاعم والكافيهات (Maestro EGP)

## المشكلة الأصلية
بناء نظام شامل لإدارة المطاعم والكافيهات يدعم:
- إدارة نقاط البيع (POS)
- إدارة الفروع المتعددة
- إدارة الموارد البشرية والرواتب
- إدارة المخزون والمشتريات
- نظام التوصيل وتتبع السائقين
- تقارير وإحصائيات ذكية
- دعم Multi-tenancy (عدة عملاء)

## الجلسة الحالية - 18 يناير 2026 (الإصدار 22)

### ✅ المهام المكتملة في هذه الجلسة:

#### 1. إصلاح مشكلة عدم ظهور المبيعات ✅
- السبب: Dashboard كان يعرض "مبيعات اليوم" فقط والطلبات من تواريخ سابقة
- الحل: إضافة نظام إحصائيات متعدد الفترات

#### 2. إحصائيات متعددة الفترات في Dashboard ✅
- **API جديد:** `/api/dashboard/stats`
- فلتر تفاعلي: اليوم / الأسبوع / الشهر / الإجمالي
- عرض: المبيعات، عدد الطلبات، متوسط الطلب، صافي الربح

#### 3. نظام إدارة اليوم والترحيل التلقائي ✅
- **APIs جديدة:**
  - `GET /api/day-management/status`
  - `POST /api/day-management/close`
  - `GET /api/day-management/history`
  - `POST /api/day-management/send-report` (جديد)
  - `GET /api/day-management/report-preview` (جديد)
- إغلاق تلقائي للورديات بعد 24 ساعة
- نافذة إدارة اليوم مع معلومات مفصلة

#### 4. تقرير يومي تلقائي عبر البريد ✅
- قالب HTML احترافي للتقرير
- إحصائيات مفصلة لكل فرع
- زر إرسال في نافذة إدارة اليوم

#### 5. طباعة إيصال خصم الموظف ✅
- زر "طباعة" في جدول الخصومات (صفحة HR)
- إيصال مفصل مع توقيع الموظف والمسؤول

#### 6. إعادة هيكلة Backend (جزئية) ✅
تم إنشاء بنية ملفات منظمة:
```
/app/backend/
├── routes/
│   ├── __init__.py
│   ├── auth.py         # المصادقة
│   ├── users.py        # المستخدمين
│   ├── hr.py           # الموارد البشرية
│   ├── branches.py     # الفروع
│   └── products.py     # المنتجات والفئات
├── services/
│   ├── __init__.py
│   └── email_service.py  # خدمة البريد
├── models/
│   ├── schemas.py      # جميع Pydantic models
│   └── enums.py        # الثوابت
├── core/
│   ├── database.py     # اتصال قاعدة البيانات
│   └── config.py       # الإعدادات
└── utils/
    ├── auth.py         # وظائف المصادقة
    └── helpers.py      # وظائف مساعدة
```

**ملاحظة مهمة:** الملفات الجديدة جاهزة للاستخدام، لكن `server.py` لا يزال يحتوي على جميع الـ APIs حالياً. الهجرة الكاملة ستتم تدريجياً.
